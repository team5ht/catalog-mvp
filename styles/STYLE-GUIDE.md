# Стили проекта: актуальное состояние

Этот файл описывает текущую структуру CSS в `styles/` и порядок подключения в HTML.

## 1) Состав папки `styles/`

```text
styles/
  components/ (пока пусто, резерв под компонентные стили)
  pages/ (пока пусто, резерв под page-level стили)
  tokens.css
  ui.css
  pages.css
  STYLE-GUIDE.md (этот файл)
```

## 2) Порядок подключения CSS в HTML

Все страницы подключают стили вручную, без сборщика. Порядок критичен:

1. `styles/tokens.css`
2. `styles/ui.css`
3. `styles/pages.css`

## 3) Назначение файлов

- `styles/tokens.css` — канонический источник токенов (CSS variables).
- `styles/ui.css` — общий UI‑kit: базовая типографика и layout, кнопки/чипы, карточки, карусели, нижняя навигация.
- `styles/pages.css` — page‑specific раскладки для главной, каталога, авторизации, материала, аккаунта.
- `styles/components/` — резерв под компонентные модули (сейчас пусто).
- `styles/pages/` — резерв под page‑level модули (сейчас пусто).

## 4) Примечания

- Сборщика нет, поэтому порядок подключения в HTML важен.
- Базовые utility‑классы для ошибок загрузки (`.load-error`) живут в `styles/ui.css`.
