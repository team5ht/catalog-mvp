# Структура стилей

Файлы в папке `styles/` разделены по уровням переиспользования.

- `tokens.css` — системные переменные цветов, теней и базовых размеров. Подключается первым на каждой странице.
- `base.css` — глобальная типографика и отступы макета.
- `components/` — переиспользуемые блоки. Здесь находятся кнопки, карточки, навигация и контейнеры со скроллом. Внутри каждого файла правила упорядочены: сначала базовый блок, затем элементы, после — модификаторы и медиа-запросы.
- `pages/` — стили конкретных страниц (каталог, карточка материала, авторизация). Они подключаются в дополнение к общим компонентам.

## Как добавлять новые стили

1. **Определите уровень.** Повторно используемые элементы помещаются в `components/`, уникальные страницы — в `pages/`.
2. **Следуйте БЭМ.** Для новых блоков используйте классы вида `.block`, `.block__element`, `.block--modifier`.
3. **Сохраняйте порядок секций.** Комментарии `/* === ... === */` помогают быстро ориентироваться внутри файла — добавляйте аналогичные, если расширяете стили.
4. **Подключайте файлы явно.** В HTML-страницах стили подключаются в порядке: `tokens.css` → `base.css` → компоненты → страничные стили.

Такой подход упрощает поиск нужных правил и снижает вероятность конфликтов между страницами.
