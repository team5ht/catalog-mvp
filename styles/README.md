# Структура стилей

Файлы в `styles/` разделены по уровням переиспользования и подключаются в порядке: `tokens.css` -> `base.css` -> компоненты -> страничные стили.

- `tokens.css` - системные переменные цветов, радиусов, теней и размеров нижней навигации (`--bottom-nav-*`). Новые компоненты опираются на эти токены.
- `base.css` - базовая типографика и отступы. Задает паддинги `main` с учетом `--bottom-nav-offset`/safe-area, общие контейнеры (`.app-shell`, `.app-container`) и баннер на главной.
- `components/` - переиспользуемые блоки:
  - `buttons.css` - два набора кнопок: `.btn*` (старые CTA/кнопка скачивания) и `.button*` (новые кнопки форм). Старайтесь не смешивать их на новых макетах.
  - `cards.css`, `scroll-containers.css` - карточки материалов и карусели.
  - `navigation.css` - нижняя навигация, toast авторизации и текстовая кнопка выхода (`data-auth-logout`) с учетом safe-area.
- `pages/` - стили конкретных страниц: `catalog.css`, `material.css`, `login.css` (общие для всех auth-страниц и легаси `login.html`). Если добавляется новая страница, создавайте отдельный файл тут.

## Как добавлять новые стили

1. Определите уровень: общие блоки - в `components/`, уникальные экраны - в `pages/`.
2. Используйте токены из `tokens.css` и существующие контейнеры вместо жестких значений.
3. Соблюдайте порядок подключения (tokens -> base -> components -> pages) и следите, чтобы отступы не перекрывали нижнюю навигацию (`--bottom-nav-offset`).
4. Подключайте новые файлы явно в HTML и при необходимости добавляйте их в `sw.js#urlsToCache` для оффлайна.
