# Стили проекта: актуальное состояние

Этот файл описывает текущую структуру CSS в `styles/` и порядок подключения в HTML.

## 1) Состав папки `styles/`

```text
styles/
  components/ (пока пусто, резерв под компонентные стили)
  pages/ (пока пусто, резерв под page-level стили)
  tokens.css
  ui.css
  pages.css
  DESIGN-CODE.md
  README.md (этот файл)
```

## 2) Порядок подключения CSS в HTML

Все страницы подключают стили вручную, без сборщика. Порядок критичен:

1. `styles/tokens.css`
2. `styles/ui.css`
3. `styles/pages.css`

## 3) Назначение файлов

- `styles/tokens.css` — единственный канонический источник токенов (CSS variables).
- `styles/ui.css` — UI‑kit: базовая типографика/контейнеры, компоненты, навигация, карточки, карусели.
- `styles/pages.css` — минимальные page‑specific раскладки (главная, каталог, авторизация, материал, аккаунт).
- `styles/components/` — зарезервировано под компонентные модули (сейчас пусто).
- `styles/pages/` — зарезервировано под page‑level модули (сейчас пусто).

## 4) Примечания

- Темы и переопределения удалены.
- В CSS нет неописанных переменных.
- `.load-error` имеет базовый стиль.
