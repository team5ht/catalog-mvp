# Структура стилей

Файлы в `styles/` разделены по уровням переиспользования и подключаются в порядке: `tokens.css` -> `base.css` -> компоненты -> страничные стили.

- `tokens.css` - палитра (primary + поверхности), текстовые и семантические цвета, набор радиусов/теней, типоразмеры, веса, отступы и токены нижней навигации (`--bottom-nav-*`).
- `base.css` — базовая типографика (system stack), фон приложения, контейнеры `.app-shell`/`.app-container`, общие заголовки и карточки, паддинги `main` с учетом `--bottom-nav-offset` и safe-area, баннер на главной.
- `components/` — переиспользуемые блоки:
  - `buttons.css` — два набора кнопок: `.btn*` (старые CTA/скачивание) и `.button*` (формы). Плюс чипы `.chip`/`.chip--interactive`/`.chip--tag` с активным состоянием. Не смешивайте системы на новых макетах.
  - `cards.css` — карточки материала для карусели с fallback при отсутствии `aspect-ratio`.
  - `scroll-containers.css` — горизонтальные карусели и скролл категорий без видимых скроллбаров; расширенные чипы для каталога.
  - `navigation.css` - нижняя навигация с safe-area, активными/hover-состояниями и классом `is-loading` для скрытия кнопки аккаунта.
- `pages/` — стили конкретных экранов: `catalog.css` (лента и поле поиска), `material.css` (картинка, кнопка назад, блок скачивания, теги), `login.css` (градиентный фон и карточка формы). Дизайн-принципы вынесены отдельно в `styles/DESIGN-CODE.md`. Если добавляется новая страница, создавайте отдельный файл здесь и подключайте его явно.

## Как добавлять новые стили

1. Определите уровень: общие блоки - в `components/`, уникальные экраны - в `pages/`.
2. Используйте токены из `tokens.css` и существующие контейнеры вместо жестких значений.
3. Соблюдайте порядок подключения (tokens -> base -> components -> pages) и следите, чтобы отступы не перекрывали нижнюю навигацию (`--bottom-nav-offset`).
4. Подключайте новые файлы явно в HTML и при необходимости добавляйте их в `sw.js#urlsToCache` для оффлайна.
